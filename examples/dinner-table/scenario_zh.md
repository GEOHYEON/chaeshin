## 1. 背景：工具定义

"晚餐准备"智能体可用的工具列表：

| 工具名 | 说明 | 输入示例 | 输出示例 |
|--------|------|----------|----------|
| `查冰箱` | 获取冰箱中的食材列表 | `{}` | `{食材: ["猪肉","豆腐","泡菜","鸡蛋","菠菜",...]}` |
| `过敏检查` | 检查家人过敏情况 | `{家人: ["妈妈","爸爸","孩子"]}` | `{结果: {孩子: ["虾"]}}` |
| `推荐菜单` | 根据食材+人数推荐菜单 | `{食材: [...], 人数: 3, 排除: ["虾"]}` | `{菜单: {汤: "大酱汤", 主菜: "辣炒猪肉", 配菜: ["菠菜拌菜","鸡蛋卷","泡菜"]}}` |
| `查食谱` | 获取特定菜品的食谱 | `{菜品: "辣炒猪肉"}` | `{步骤: [...], 时间: "25分钟", 难度: "中"}` |
| `备料` | 洗、切、焯等食材处理 | `{食材: "菠菜", 方法: "焯水"}` | `{状态: "完成", 耗时: "5分钟"}` |
| `烹饪` | 实际烹调（炒/煮/烤等） | `{菜品: "辣炒猪肉", 工序: "炒", 火力: "大火"}` | `{状态: "完成", 味道: "OK"}` |
| `尝味` | 检查和调整味道 | `{菜品: "大酱汤", 检查: ["咸度","鲜味"]}` | `{味道: "OK"}` 或 `{味道: "偏淡", 建议: "加盐"}` |
| `装盘` | 盛入碗碟 | `{菜品列表: [...], 人数: 3}` | `{状态: "完成"}` |
| `摆桌` | 在餐桌上摆放所有菜品 | `{菜单列表: [...], 人数: 3}` | `{布局: "完成", 缺少: []}` |
| `设定时器` | 烹饪计时 | `{时间: "20分钟", 目标: "大酱汤"}` | `{状态: "计时开始"}` |

## 2. 场景 A：成功案例

### 用户请求

```
"今晚准备一桌饭菜。3个人，孩子对虾过敏。"
```

---

### 第1步：Retrieve（搜索过去的案例）

智能体首先做的是 **搜索过去是否有类似的请求**。

```python
chaeshin_retrieve(
    query="准备晚餐一桌 3人 有过敏",
    category="家常菜",
    keywords="晚餐,一桌,3人,过敏,家常菜"
)
```

**搜索结果：**

```
cases: []           # 第一次，没有匹配的案例
warnings: []        # 也没有失败记录
```

> **判断：** 没有匹配 → 需要从头创建新的图。

---

### 第2步：Decompose（层级分解）

Chaeshin v2 将复杂请求分解为 **3个层级**。

```
L3（策略）："准备一桌晚餐"
│
├── L2（模式）："确定菜单"
│   ├── L1（执行）：查冰箱
│   ├── L1（执行）：过敏检查
│   └── L1（执行）：推荐菜单
│
├── L2（模式）："做汤"
│   ├── L1（执行）：查食谱 → 备料 → 烹饪 → 尝味
│   └── L1（执行）：设定时器
│
├── L2（模式）："做主菜"
│   ├── L1（执行）：查食谱 → 备料 → 烹饪
│   └── L1（执行）：尝味
│
├── L2（模式）："准备配菜"
│   ├── L1（执行）：备料(菠菜) → 烹饪(拌菜)
│   ├── L1（执行）：备料(鸡蛋) → 烹饪(鸡蛋卷)
│   └── L1（执行）：装盘(泡菜) ← 已有配菜
│
└── L2（模式）："摆桌"
    └── L1（执行）：装盘(全部) → 摆桌
```

**难度计算：** `depth = 3` → difficulty = 3（满足 Chaeshin 触发条件 `difficulty >= 2`）

## 4. 两个场景对比

| 项目 | 场景 A（成功） | 场景 B（失败→恢复） |
|------|---------------|-------------------|
| **请求** | 一桌晚餐 3人 | 一桌晚餐 3人（食材不足） |
| **Retrieve** | 无匹配（首次） | 现有案例匹配（0.78） |
| **执行节点** | 13个（全部） | 13个 → 失败 → 缩减为5个 |
| **循环** | 尝味1次（大酱汤） | 无 |
| **升级** | 无 | L1→L2→用户 |
| **最终结果** | 一桌（汤+主菜+配菜3种） | 简餐（炒饭+鸡蛋卷） |
| **满意度** | 0.92 | 0.70 |
| **保存** | 成功案例1个 | 失败警告1个 + 简餐成功1个 |
| **学习效果** | 下次一桌请求可复用 | 食材不足时避免套用一桌 |

---

## 关键要点

1. **Retrieve First**：一定先搜索过去的案例。相似度 > 0.7 就复用。
2. **层级分解**：复杂请求分为 L3→L2→L1。每层独立执行。
3. **Edge Condition**：节点输出决定下一步路径。`味道 == 偏淡` → 循环；`味道 == OK` → 继续。
4. **升级机制**：L1 失败 → L1 重新规划 → L2 → 用户。逐级升级，绝不跳级。
5. **反模式**：失败也保存，避免重蹈覆辙。
6. **并行执行**：独立任务并行处理（查冰箱+过敏检查、汤+主菜+配菜同时烹饪）。
